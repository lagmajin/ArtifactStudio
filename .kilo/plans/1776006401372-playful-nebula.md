# Plan: Implement Message Importance Highlighting in AI Client

## Goal
Add functionality to detect and visually emphasize important messages in the AI chat client, improving user experience by drawing attention to critical information like errors, warnings, or key responses.

## Current State Analysis
- **AI Client Components**: `ArtifactAICloudWidget` manages the UI, `CloudChatBubble` displays individual messages with role-based styling (User: blue right-aligned, Assistant: dark left-aligned, System: brown center-aligned).
- **Message Flow**: Messages are added via `appendTranscriptMessage()` and displayed as plain text in QLabel within CloudChatBubble.
- **Existing Styling**: Uses QPalette and custom drawing, avoiding QtCSS as per project guidelines.

## Proposed Solution

### 1. Importance Detection Logic
Implement a simple keyword-based detection system in `ArtifactAICloudWidget`:
- Scan message text for keywords indicating importance: "error", "failed", "warning", "critical", "bug", "issue", "exception", "alert", "urgent", "important".
- Add optional sentiment heuristics (count negative words like "problem", "fail").
- Future enhancement: Integrate API metadata (e.g., error finish reasons, high token counts) if exposed by `AIClient`.

### 2. UI Enhancement
Extend `CloudChatBubble` to support importance levels:
- Add `Importance` enum: Normal, Important, Critical.
- Modify `paintEvent()` to apply visual emphasis:
  - **Important**: Red border/border color change, bold font.
  - **Critical**: Warning icon, red gradient background, larger font.
- Ensure compatibility with existing role-based styling.

### 3. Implementation Steps
1. Update `CloudChatBubble` class:
   - Add importance enum and setter method.
   - Modify paint logic for emphasis without breaking existing styles.
2. Add detection function in `ArtifactAICloudWidget`:
   - `Importance detectImportance(const QString& text)` method.
   - Integrate into `appendTranscriptMessage()` and `replaceLastAssistantMessage()`.
3. Test with sample messages and verify styling.
4. Run lint and typecheck commands post-implementation.

### 4. Tradeoffs and Alternatives
- **Keyword vs. Advanced Detection**: Starting with keyword matching for simplicity; can upgrade to sentiment analysis or LLM-based detection if needed.
- **Visual Emphasis**: Border/color changes preferred over icons to maintain performance; can add icons for critical messages.
- **Scope**: Focus on Assistant messages initially; extend to all roles if requested.

## Questions for User
- What constitutes an "important" message in your context? (e.g., specific keywords, error patterns, or other criteria?)
- What type of visual emphasis do you prefer? (e.g., color changes, icons, animations?)
- Should this apply to all message roles or just AI responses?

## Risks
- Performance impact if detection is too complex; mitigate by keeping it lightweight.
- UI conflicts with existing styling; test thoroughly.
- False positives in keyword detection; refine keyword list based on feedback.
