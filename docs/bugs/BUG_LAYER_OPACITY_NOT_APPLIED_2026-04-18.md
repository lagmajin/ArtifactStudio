# 調査報告書: レイヤー不透明度 反映されない問題

**調査日**: 2026-04-18
**対象ファイル**: コンポジションレンダーコントローラ
**重大度**: 🟥 致命的

---

## 問題の概要

プロパティウィジェットでレイヤーの不透明度を変更しても、コンポジションエディタ上では一切反映されない。
値は正常に保存され、プロパティ上は正しい値が表示されるが、画面上は常に100%不透明のまま。

---

## 🔍 根本原因

### 各レイヤー側の実装 ✅ 正常

全てのレイヤークラスの `draw()` メソッドでは、正しく不透明度の乗算が実装されている:

```cpp
// 全レイヤー共通の実装
renderer->drawImageTransformed(
    transform, 
    image, 
    this->opacity() * weight);
```

✅ 不透明度プロパティは正常に動作
✅ キーフレームアニメーションも正常に動作
✅ 各レイヤー内部では正しくアルファ値が乗算されている

---

### コンポジションコントローラ側 ❌ バグ

レイヤーが正しくアルファを適用して描画した後、最終合成ステップで **常に alpha=1.0 として上書き合成** されている。

```cpp
// 問題の箇所
for (auto& layer : layers) {
    // レイヤーは自分自身で不透明度を適用済み
    layer->draw(renderer);
    
    // しかしこの後、コントローラ側で
    // 常に不透明度 1.0 として合成が実行される
    compositeLayerBuffer(layer, 1.0f); // ← ここで値が無視される
}
```

レイヤー側でいくら不透明度を適用しても、最終段階で完全に上書きされてしまっている。

---

## 📊 影響範囲

✅ 全てのレイヤータイプが影響を受ける
✅ 不透明度アニメーションも全く動作しない
✅ ブレンドモードは正常に動作する
✅ マスクは正常に動作する

不透明度だけが完全に機能していない状態。

---

## ✅ 修正方法

### 修正に必要な変更点はたった1行

```cpp
// 修正前
compositeLayerBuffer(layer, 1.0f);

// 修正後
compositeLayerBuffer(layer, layer->opacity());
```

この1行を変更するだけで、全てのレイヤーの不透明度が正常に反映されるようになります。

---

## 💡 補足

このバグは非常に特徴的です:

- 各所で何重にもチェックと実装がされている
- 最後の最後の一番最後のステップだけが間違っている
- 誰もレイヤー側を疑って何度も見直している
- 問題は常に一番見えない箇所に存在する

実際の修正は5秒で終わります。
