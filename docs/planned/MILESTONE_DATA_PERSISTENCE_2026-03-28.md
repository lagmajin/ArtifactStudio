## Phase 構成

### Phase 1: 原子操作的保存

- 目的:
  - 保存中のクラッシュからデータを保護

- 作業項目:
  - テンポラリファイルへの保存
  - 完了後のリネーム
  - バックアップの保持（3 世代）

- 完了条件:
  - クラッシュしても前回保存分は保護
  - バックアップから復元可能

- 実装案:
  ```cpp
  bool ArtifactProject::save() {
      // 1. テンポラリファイルに保存
      QString tempPath = filePath_ + ".tmp";
      if (!saveToFile(tempPath)) {
          return false;
      }
      
      // 2. バックアップを作成
      if (QFile::exists(filePath_)) {
          QString backupPath = filePath_ + ".bak~1";
          QFile::copy(filePath_, backupPath);
      }
      
      // 3. アトミックにリネーム
      return QFile::rename(tempPath, filePath_);
  }
  ```

### Phase 2: バージョン管理

- 目的:
  - 互換性の保証

- 作業項目:
  - ファイルバージョンの記録
  - バージョンチェック
  - 移行スクリプト

- 完了条件:
  - 旧バージョンファイルを開ける
  - 新バージョンへ自動移行

- 実装案:
  ```cpp
  // プロジェクトファイル構造
  {
      "version": "1.2.0",
      "minVersion": "1.0.0",
      "data": { ... }
  }
  
  // バージョンチェック
  bool ArtifactProject::load(const QString& path) {
      QJsonDocument doc = loadJson(path);
      QString version = doc["version"].toString();
      
      if (!isCompatibleVersion(version)) {
          // 移行スクリプトを実行
          return migrate(doc, version);
      }
      
      return loadFromJson(doc);
  }
  ```

### Phase 3: 参照切れ検出

- 目的:
  - アセット参照の整合性保証

- 作業項目:
  - 保存時に参照チェック
  - 自動修復機能
  - エラーレポート

- 完了条件:
  - 参照切れを保存前に検出
  - ユーザーに修復を提案

### Phase 4: バリデーション強化

- 目的:
  - データ品質の向上

- 作業項目:
  - スキーマ定義
  - 循環参照検出
  - 必須フィールドチェック

- 完了条件:
  - 不正なデータは保存されない
  - エラーメッセージが明確

## 技術的課題

### 1. 増分保存の実装

**課題:**
- どの部分が変更されたかを追跡

**解決案:**
- コマンドパターンでの履歴管理
- ダーティフラグの伝播
- 差分のシリアライズ

### 2. バージョン移行

**課題:**
- 複雑なデータ構造の移行

**解決案:**
- 段階的な移行
- デフォルト値の補完
- 削除フィールドの処理

### 3. パフォーマンス

**課題:**
- バリデーションのコスト

**解決案:**
- 非同期バリデーション
- キャッシュの活用
- 遅延評価
