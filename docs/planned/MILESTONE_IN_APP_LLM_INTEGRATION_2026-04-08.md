## ■ 概要
ArtifactStudio エディタ内に llama.cpp ベースのローカル LLM 実行環境を統合し、
ユーザーが外部サービス無しで AI アシスト機能を利用可能にする。

GPU 上でモデルを常駐実行し、エディタ操作とシームレスに連携する。

## ■ 技術的な対応事項

### ✅ 解決済み課題
- Windows 上での日本語文字化け: 修正手順確定済み
- MSVC コンパイル: パッチ適用方法確立

### ⚠️ 既知の課題
- llama.dll のサイズ: ~120MB
- 初回ロード時間: RTX 4070 で 約2.5秒
- 同時実行スレッド数の調整

---

## ■ アーキテクチャ案
```
ArtifactStudio UI
    ↓
ArtifactLLMBridge (Qt スレッド)
    ↓
llama.cpp C API ラッパー
    ↓
CUDA / DirectML / CPU バックエンド
```

- メインスレッドをブロックしない非同期設計
- トークン生成毎のストリーミング出力
- メモリ使用量の動的監視と閾値制御

## ■ 優先実装機能
1.  **選択したレイヤーに対する操作指示**
    > 「このテキストを少し明るくして」
    > 「3秒間フェードインするように」

2.  **タイムライン一括編集**
    > 「全部のクリップの間に 0.5秒 の余白を入れて」

3.  **エフェクト提案**
    > 「この動画に合うエフェクトを3つ提案して」

4.  **エラー解決アシスト**
    > レンダリング失敗時に原因と解決方法を提示

## ■ リスクと緩和策
| リスク | 発生確率 | 影響度 | 緩和策 |
|--------|----------|--------|--------|
| モデルサイズが大きすぎる | 高 | 中 | デフォルト 3B / 7B 選択式、オンデマンドダウンロード |
| レンダリング性能への影響 | 中 | 高 | レンダリング実行時は LLM を一時停止 |
| 出力品質が低い | 中 | 中 | ドメイン特化ファインチューンモデルを後日提供 |
