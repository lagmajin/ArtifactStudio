# M12 Polish and Stability (2026-03-17)

「機能するプロトタイプ」から「日常使えるツール」へ。
ユーザーがストレスなく作業を続けられる品質と信頼性を確立する。

## Goal

- 主要な制作パス（コンポ作成→レイヤー配置→エフェクト→書き出し）が破綻なく行える
- UI 操作が直感的で、迷いにくい
- クラッシュやデータ消失のリスクが低く、万が一の際も回復できる
- 低・中スペック環境でも実用的な速度で動作する

## Definition Of Done

- **制作フローの安定性**
  - 新規プロジェクトから静止画シーケンス書き出しまでがエラーなく完了する
  - 主要な編集操作（追加、削除、移動、rename）が意図通りに動作する
  - undo/redo が主要な編集操作で機能する

- **UI/UX の品質**
  - Timeline のプレイヘッド操作が滑らかで、不自然な挙動がない
  - Project View と Timeline の選択同期が一貫している
  - Inspector で Effect の追加・削除・並べ替えが直感的に行える
  - 各パネルの空状態に適切なガイダンスが表示される

- **安定性と回復性**
  - 頻繁な操作でクラッシュしない（連続 30 分操作テストをパス）
  - 保存・読込でデータが欠損しない
  - 異常終了時に前回状態の回復案内が表示される

- **パフォーマンス**
  - 10 レイヤー程度のコンポジションでプレビューが 15fps 以上維持される
  - 50 レイヤーまで UI 操作が著しく重くならない
  - 初回起動から 5 秒以内に操作可能になる

## Work Packages

### 1. Timeline Refinement (M-TL-FINAL)
**目的**: タイムライン操作の直感性と安定性を最終調整する

**対象**:
- プレイヘッドの追従・スナッピング・不感帯
- ホイール操作（ズーム・スクロール）の自然さ
- レイヤーのドラッグ移動・トリミング・選択範囲
- 行の高さ・フォント・アイコンの視認性
- 空状態のガイド表示

**完了条件**:
- プレイヘッドが意図しない位置に飛ばない
- ズーム操作がスムーズで、拡大・縮小の境界で止まらない
- レイヤーの境界スナップがオン/オフで明確に切り替わる
- 100 レイヤーまでスクロールがカクつかない

---

### 2. Inspector & Effect Stack (M-FX-INSPECTOR)
**目的**: Inspector でのエフェクト操作を直感的にする

**対象**:
- Effect の追加ダイアログ（検索・カテゴリ・プレビュー）
- スタックの順序変更（ドラッグ・上下ボタン）
- 各エフェクトのパラメータ編集 UI
- エフェクトのオン/オフ切り替え
- 空状態のメッセージ

**完了条件**:
- Inspector から Effect を検索して即座に追加できる
- ドラッグで順序を変更し、即座にプレビューに反映される
- 各パラメータがスライダー・数値入力・カラーピッカーで編集できる
- エフェクトのオン/オフで即座に差分を確認できる

---

### 3. Undo/Redo Framework (M-UNDO-1)
**目的**: 主要な編集操作で undo/redo を機能させる

**対象**:
- コマンドパターンの実装
- 編集履歴の管理（スタック・最大件数・シリアライズ）
- 対応コマンド：レイヤー追加/削除/移動/rename、エフェクト追加/削除、パラメータ変更
- UI での表示（メニュー・ツールバー・ショートカット）

**完了条件**:
- Ctrl+Z / Ctrl+Y で 10 段階以上の undo/redo が機能する
- 履歴パネルで過去の操作を確認できる
- ファイル保存・読込後も undo スタックが維持される（オプション）

---

### 4. Crash Recovery & Diagnostics (M-QA-RECOVERY)
**目的**: クラッシュ時の回復と原因特定を容易にする

**対象**:
- 例外ハンドラとスタックトレース収集
- 簡易ダンプファイルの自動保存
- 重要オブジェクト（Project/Composition/Asset）のスナップショット
- 次回起動時の回復ダイアログ
- ログファイルの整備（出力先・ローテーション・レベル）

**完了条件**:
- 異常終了時にスタックトレースがログに残る
- 次回起動時に「前回異常終了しました」と表示され、回復オプションが提示される
- 主要なクラッシュシナリオ（不正なファイル読込・メモリ不足）でセーフモード起動ができる

---

### 5. Save/Load Integrity (M-AS-SAVELOAD)
**目的**: プロジェクトファイルの保存・読込でデータが欠損しない

**対象**:
- Composition/Layer/Effect のシリアライズ完全実装
- 破損ファイルの検出とエラーメッセージ
- 保存中の自動バックアップ（.autosave）
- 読み込み時のバージョン互換処理

**完了条件**:
- 全レイヤータイプ・エフェクト・マ스크・キーフレームが保存/読込できる
- 保存中にクラッシュしても、次回起動時に autosave から回復できる
- バージョンアップ後のファイルも旧バージョンで読込失敗時に明確なエラーが出る

---

### 6. Performance Baseline (M-PERF-1)
**目的**: 実用的な動作速度を確保する

**対象**:
- 描画の最適化（不要な再描画の排除）
- タイムラインの仮想化（行の再利用）
- プロジェクトビューのキャッシュ
- 画像読み込みの非同期化
- メモリ使用量の監視と警告

**完了条件**:
- 10 レイヤー：プレビュー 15fps 以上、UI 操作 30fps 以上
- 50 レイヤー：UI 操作が著しく重くならない（100ms 以内応答）
- 100 レイヤー：スクロール・ズームが操作可能
- メモリ使用量が 2GB を超える前に警告が表示される

---

### 7. Onboarding & Empty States (M-UX-ONBOARD)
**目的**: 初回ユーザーが迷わずに済む導線を作る

**対象**:
- 初回起動時のチュートリアル（オプション）
- 各パネルの空状態に「次のアクション」を表示
- ヘルプメニューとショートカット一覧
- サンプルプロジェクトの同梱

**完了条件**:
- 新規プロジェクト作成時に「Solid を追加」「Import する」のクイックアクションが出る
- Timeline 空時に「レイヤーを追加するには」というガイドが出る
- ヘルプ → ショートカット一覧で主要操作を確認できる
- サンプルプロジェクトで基本的な機能を確認できる

---

## Suggested Order

1. **M-AS-SAVELOAD**（データ消失防止が最優先）
2. **M-QA-RECOVERY**（クラッシュ対策）
3. **M-TL-FINAL**（タイムラインの安定性）
4. **M-FX-INSPECTOR**（エフェクト操作の完善）
5. **M-UNDO-1**（編集の安心感）
6. **M-PERF-1**（パフォーマンス底上げ）
7. **M-UX-ONBOARD**（ユーザー体験の仕上げ）

## Metrics

| 指標 | 目標値 | 測定方法 |
|------|--------|----------|
| クラッシュフリー時間 | 30 分以上 | 手動テスト |
| 主要操作の応答時間 | 100ms 以内 | プロファイラ |
| プレビュー fps（10L） | 15fps 以上 | フレーム計測 |
| undo 段階数 | 10 以上 | 手動テスト |
| 保存可能データカバレッジ | 100% | 網羅テスト |
| 初回操作完了率 | 80% 以上 | ユーザテスト |
