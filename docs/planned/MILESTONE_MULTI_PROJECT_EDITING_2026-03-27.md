# マルチプロジェクト編集 Milestone

**作成日:** 2026-03-27  
**ステータス:** 計画中  
**関連コンポーネント:** ArtifactProjectManager, ArtifactMainWindow, ArtifactProjectService

---

## 概要

複数のプロジェクトファイルを同時に開いて編集・参照できる機能を実装する。

After Effects の「複数プロジェクト同時編集」や、Blender の「リンク/アペンド」機能に相当する機能。

---

## Goal

- 3 つ以上のプロジェクトを同時に開く
- プロジェクトタブで素早く切り替え
- 他プロジェクトのアセットを参照・インポート
- レイヤー/コンポジションを別プロジェクトにコピー
- 編集中のプロジェクトを明確に表示

## Background

現在の ArtifactStudio は、1 つのプロジェクトファイルしか開けない。

これにより以下の制限がある：

1. **アセットの使い回しが困難** - 毎回インポートし直し
2. **比較作業ができない** - 2 つのプロジェクトを並べて比較
3. **コピー＆ペースト不可** - レイヤーを別プロジェクトに移動

この milestone は、これらの制限を解消する。

## Phases

### Phase 1: マルチプロジェクト基盤

- 目的:
  - 複数プロジェクトの同時読み込み
  - プロジェクトタブでの切り替え

- 作業項目:
  - `ArtifactProjectManager` のマルチプロジェクト対応
  - プロジェクトタブ UI の実装
  - アクティブプロジェクトの切り替えロジック
  - 各プロジェクトの独立した状態管理

- 完了条件:
  - 3 つ以上のプロジェクトを同時に開ける
  - タブクリックでプロジェクトを切り替え
  - 各プロジェクトが独立して編集可能

- UI 案:
  ```
  ┌─────────────────────────────────────────────────┐
  │ [ProjectA] [ProjectB] [ProjectC] [+]          │ ← プロジェクトタブ

  │                                                 │

### Phase 2: クロスプロジェクト参照

- 目的:

  - 外部参照の解決とパス管理

- 作業項目:
  - アセットブラウザに「他プロジェクト」セクション追加
  - 相対パス/絶対パスの解決ロジック
  - リンク切れ検出と警告
  - アセットのプレビュー表示

- 完了条件:
  - 他プロジェクトのアセットを一覧表示
  - ドラッグ＆ドロップでインポート
  - リンク切れを警告

- UI 案:
  ```
  ┌─────────────────────────────────────────────────┐
  │ アセットブラウザ                                │
  ├─────────────────────────────────────────────────┤
  │ ▼ 現在のプロジェクト                            │
  │   📁 Images                                     │
  │   📁 Videos                                     │

  │     📁 Images                                   │

  ```

### Phase 3: クロスプロジェクト編集

- 目的:
  - レイヤーを別プロジェクトにコピー
  - コンポジション単位での移動

- 作業項目:
  - クロスプロジェクトコピー＆ペースト
  - 依存関係の解決
  - エフェクト/マテリアルのコピー
  - タイムラインキーフレームのコピー

- 完了条件:
  - レイヤーを別プロジェクトにコピー可能
  - 依存関係が正しく解決される
  - キーフレームが保持される

- 実装フロー:
  ```
  1. レイヤーをコピー（Ctrl+C）
     ↓
  2. 他プロジェクトに切り替え
     ↓
  3. ペースト（Ctrl+V）
     ↓
  4. 依存アセットを自動コピー or リンク
     ↓
  5. 完了
  ```

### Phase 4: リンクアセット機能

- 目的:
  - 元プロジェクトとのリンクを維持
  - 更新の自動/手動同期

- 作業項目:
  - リンクアセットの作成
  - 元プロジェクトの監視
  - 更新検出と同期オプション

- 完了条件:
  - リンクアセットとしてインポート可能
  - 元の変更を検出して通知

- リンク状態:

  ```

## 技術的課題

### 1. メモリ管理

複数プロジェクトを同時に開くため、メモリ使用量が増加する。

**対策:**
- 非アクティブプロジェクトのレイジー読み込み
- アセットキャッシュの共有
- 使用頻度の低いデータのディスク退避

### 2. パス解決

プロジェクト間の相対パス解決が複雑になる。

**対策:**
- 絶対パスと相対パスの両方を保持
- プロジェクトルートからの相対パスを標準化
- リンク切れ検出と修復機能

### 3. 状態管理

各プロジェクトの独立した状態を管理する必要がある。

**対策:**
- `ProjectContext` クラスで状態をカプセル化
- アクティブプロジェクトのみ UI を更新
- 他プロジェクトはバックグラウンドで管理

## UI/UX 設計

### プロジェクトタブ

```
┌─────────────────────────────────────────────────────────┐
│ [📄 ProjectA ✕] [📄 ProjectB ✕] [📄 ProjectC ✕] [+]  │
└─────────────────────────────────────────────────────────┘
```

- **左クリック** - プロジェクト切り替え
- **✕ ボタン** - プロジェクトを閉じる
- **[+] ボタン** - 新しいプロジェクトを開く
- **ダブルクリック** - プロジェクト名を編集
- **右クリック** - コンテキストメニュー（名前変更、パス表示など）

### ドラッグ＆ドロップ

```
ProjectA のレイヤー → ProjectB にドロップ
         ↓
┌─────────────────────────────────┐
│ どのようにインポートしますか？  │
├─────────────────────────────────┤
│ ○ コピー（独立したレイヤー）   │
│ ○ リンク（元と同期）           │
│ ○ インスタンス（参照のみ）     │
├─────────────────────────────────┤
│      [キャンセル] [OK]          │
└─────────────────────────────────┘
```
