## Phase 構成

### Phase 1: エフェクトプラグイン基盤

- 目的:
  - サードパーティ製エフェクトを可能に

- 作業項目:
  - プラグインインターフェース定義
  - DLL/共有ライブラリの読み込み
  - パラメータシリアライズ

- 完了条件:
  - 外部エフェクトを読み込み可能
  - パラメータ UI が自動生成

- 実装案:
  ```cpp
  // プラグインインターフェース
  class IEffectPlugin {
  public:
      virtual QString name() const = 0;
      virtual QString version() const = 0;
      virtual QList<EffectParameter> parameters() const = 0;
      virtual void process(QImage& image, const QMap<QString, QVariant>& params) = 0;
  };
  
  // プラグインの読み込み
  QPluginLoader loader(path);
  QObject* plugin = loader.instance();
  IEffectPlugin* effect = qobject_cast<IEffectPlugin*>(plugin);
  
  if (effect) {
      // パラメータ UI を自動生成
      for (const auto& param : effect->parameters()) {
          createWidget(param);
      }
  }
  ```

### Phase 2: スクリプティング API

- 目的:
  - 自動化と拡張を可能に

- 作業項目:
  - Python バインディング
  - アプリケーションオブジェクトの公開
  - マクロ記録

- 完了条件:
  - Python から全操作が可能
  - マクロの記録/再生

- 実装案:
  ```python
  # Python API 例
  import artifact
  
  app = artifact.Application.instance()
  project = app.currentProject()
  comp = project.createComposition("New Comp", 1920, 1080)
  
  layer = comp.addSolidLayer(255, 0, 0, "Red Solid")
  layer.position = (960, 540)
  layer.scale = (100, 100)
  
  # キーフレーム設定
  layer.position.addKeyframe(0, (960, 540))
  layer.position.addKeyframe(30, (960, 400))
  
  # エフェクト追加
  effect = layer.addEffect("Blur")
  effect.amount = 50
  ```
