# マイルストーン: PMD ファイルインポート対応

> 2026-04-19 作成

## 目的

`ArtifactCore` の 3D モデル読み込み経路に PMD (MikuMikuDance Polygon Model Data) フォーマット対応を追加し、
既存の 3D ビューワーおよびレイヤーシステムと統合する。

---

## 背景

PMD フォーマットは MMD エコシステムで最も広く使われている 3D モデルフォーマットであり、
コンテンツ制作ワークフローにおける重要な入力形式となる。

現在の状況:
- `FileTypeDetector` は未だ `.pmd` を認識していない
- `MeshImporter` は PMD 用のデコーダを持っていない
- `Artifact3DModelViewer` はボーン・ウェイト情報を表示する仕組みが無い
- マテリアル・テクスチャ・モーフ対応の下地が不足している

---

## 方針

### 原則

1. 既存の `MeshImporter` パイプラインに追加する形で実装する
2. `ufbx` と同じインターフェイスを実装し、透過的に扱えるようにする
3. 最初は読み込み専用、編集機能は後段階で検討
4. MMD 特有のプロパティは既存のアセットシステムにマッピングする
5. 後方互換性を維持し、既存の 3D パイプラインを破壊しない

### 対象フォーマット

- `.pmd` (MMD モデルファイル v1/v2)
- 将来: `.pmx` (拡張版)

---

## Phase 1: ファイルフォーマット認識とデコーダ基盤

### 目的
PMD ファイルをシステム全体で認識し、最低限のジオメトリ読み込みを実装する。

### 作業項目
- [ ] `FileTypeDetector` に `.pmd` を `Model3D` として登録
- [ ] PMD バイナリパーサの最小実装
- [ ] 頂点・面・法線情報の読み込み
- [ ] `MeshImporter` への登録とフォールバック制御
- [ ] エラーハンドリングとバージョン判別

### 完了条件
- エクスプローラ / プロジェクトビューで PMD ファイルが 3D モデルとして表示される
- Contents Viewer で頂点情報のみのワイヤーフレーム表示ができる
- 破損ファイル読み込み時に適切なエラーが表示される

## 注意点

- PMD フォーマットは仕様書と実際のファイルの差異が大きいため、実ファイルベースで実装すること
- 文字列エンコーディングは Shift-JIS が標準である点に注意
- 頂点インデックスの重複や不正な値が非常に多いため、サニタイズ処理が必須
- 法線情報が壊れているファイルが多数存在するため、自動再計算オプションを用意する
