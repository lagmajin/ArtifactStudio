# Render / Output Feel Refinement Milestone

**作成日:** 2026-03-27  
**ステータス:** 計画中  
**関連コンポーネント:** ArtifactRenderQueueService, ArtifactRenderQueueManagerWidget, ArtifactProjectService

---

## 概要

レンダリングキューの「使いやすさ」と「失敗時の戻しやすさ」を改善する。

現在のレンダーキューは、ジョブを追加して実行するだけの単純なもの。この milestone では、以下の改善を行う：

1. **失敗からの回復** - 途中失敗からの再開を容易に
2. **可視化** - 何が重いかを明確に表示
3. **履歴管理** - 命名・履歴・再実行を容易に
4. **状態管理** - 編集中の状態を明確に表示

---

## Goal

- 失敗したジョブの再開が 1 クリックで可能
- 各フレームの処理時間が可視化される
- レンダリング履歴から簡単に再実行可能
- 編集中のジョブ名をその場で変更可能

## Background

現在のレンダーキューの問題点：

1. **失敗時の再開が面倒** - 最初からやり直し
2. **何が重いか分からない** - フレームごとの処理時間が見えない
3. **履歴管理がない** - 前回の設定を覚えていない
4. **命名が固定** - ジョブ名を後から変更できない

この milestone は、これらの「使いにくさ」を解消する。

## Phases

### Phase 1: Render Recovery（失敗からの回復）

- 目的:
  - 途中失敗からの再開を容易にする
  - 失敗フレームの特定と再レンダリング

- 作業項目:
  - **失敗フレーム検出器** - 欠損フレームを自動検出
  - **失敗フレーム一覧表示** - どのフレームが失敗したかを表示
  - **選択的再レンダリング** - 失敗フレームのみを再レンダリング
  - **整合性チェック** - 出力ファイルの欠落・破損を検出

- 完了条件:
  - 失敗したフレームを一覧表示
  - 失敗フレームのみを再レンダリング可能
  - 出力整合性チェックを実行可能

- UI 案:
  ```
  ┌─────────────────────────────────────────────────┐
  │ ジョブ 1: Scene_01.mp4                          │
  │ 状態：失敗（120/300 フレーム）                  │
  ├─────────────────────────────────────────────────┤
  │ ❌ 失敗フレーム：45, 67, 89, 123, 156           │
  │                                                 │
  │ [失敗フレームのみ再レンダリング] [最初から]    │
  └─────────────────────────────────────────────────┘
  ```

- 実装詳細:
  ```cpp
  // 失敗フレーム検出
  struct FailedFrameInfo {
      int jobId;
      int frameNumber;
      QString errorMessage;
      qint64 timestamp;
  };
  ```

## 技術的課題

### 1. 失敗フレームの検出

レンダリング中のフレームと出力済みフレームの整合性をどう取るか。

**解決案:**
- 各フレームの完了時にマーカーファイルを作成（`.frame_045.done`）
- 出力後にファイルサイズをチェック
- ジョブメタデータに進捗を記録

### 2. パフォーマンスオーバーヘッド

診断情報の収集がレンダリングを遅くしないか。

**解決案:**
- サンプリング収集（全フレームではなく 10 フレームに 1 回）
- バックグラウンドで非同期収集
- オプションで無効化可能に

### 3. 状態管理の複雑化

複数ジョブの状態をどう管理するか。

**解決案:**
- `RenderJobState` 列挙型で状態を定義
- 状態変更はシグナルで通知
- UI はシグナルを受けて更新
