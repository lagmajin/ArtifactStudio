# Render Queue Manager ギャップ分析

> 2026-04-13 作成

現状の `RenderQueueManagerWidget` を確認し、不足・改善点を整理した。

## 機能面の不足

1. **画像シーケンス出力 (PNG/TIFF/EXR連番)**
   - 現状は動画フォーマット中心で、連番書き出しの導線がない

2. **オーディオレンダリング分離**
   - 音声のみの出力や、映像+音声のミックス設定が弱い

3. **マルチコンポジション一括追加**
   - `Add` ボタンはあるが、プロジェクト内の全コンポを一括キュー投入する導線がない

4. **テンプレート/プリセットのインポート・エクスポート**
   - プリセットの保存・読み込みはあるが、ファイルとしてのエクスポート・共有ができない

5. **レンダー後の自動アクション**
   - レンダー完了後のフォルダを開く、通知、サウンド再生などがない

6. **ジョブの優先度制御**
   - キューの順序変更はあるが、優先度（High/Low）の概念がない

7. **バックグラウンドレンダリングの進捗通知**
   - ウィンドウを閉じている間の進捗が分からない（システムトレイ通知などがない）

8. **レンダー設定のコンポジション間コピー**
   - 一つのジョブの設定を別ジョブへ一括コピーする導線がない

## UI/UX面の不足

9. **サムネイルプレビュー**
   - ジョブリストにコンポジションのサムネイルがない

10. **推定残り時間表示**
    - プログレスバーはあるが、ETA（推定完了時間）がない

11. **レンダー結果のプレビュー**
    - 完了したジョブをその場でプレビュー再生する導線がない

12. **エラー詳細の展開表示**
    - `errorMessage` はあるが、スタックトレースや再試行導線が弱い

13. **ジョブのドラッグ&ドロップ並べ替え**
    - moveUp/moveDown ボタンのみで、ドラッグでの並べ替えがない

## アーキテクチャ面の不足

14. **レンダーサービスの非同期キャンセル**
    - キャンセルボタンはあるが、実際のレンダープロセスの安全な中断フローが不明

15. **レンダー設定のシリアライズ整合**
    - プロジェクト保存時にキューの状態が保存・復元されるかが不明

16. **複数エンコーダバックエンドの切替**
    - `encoderBackend` フィールドはあるが、実際の切替ロジックが未完成の可能性

## 推奨優先順位

1. 画像シーケンス出力 → 制作ワークフローの基本
2. ETA表示 → UX改善の即効性が高い
3. レンダー後の自動アクション → 実用性の壁を越える
4. マルチコンポ一括追加 → 作業効率化
5. エラー詳細の展開 → トラブルシューティング改善
6. ドラッグ&ドロップ並べ替え → 操作感向上
