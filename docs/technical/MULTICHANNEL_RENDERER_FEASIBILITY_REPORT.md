## 🔧 実際に追加が必要なもの

### 1. レンダラー側: 10行程度の追加
```cpp
// 既に存在するメソッドに2行追加するだけ
void ArtifactIRenderer::renderLayer(ArtifactAbstractLayer* layer)
{
    // 既存の RGBA 描画
    drawLayerColor(layer);
    
    // ✅ 追加するだけ
    if (multichannelEnabled_) {
        drawLayerDepth(layer);      // 既に内部的には動作している
        drawLayerNormal(layer);     // 既に内部的には動作している
        drawLayerVelocity(layer);   // モーションパスから流用可能
        drawLayerObjectId(layer);   // レイヤーIDを書き込むだけ
    }
}
```

### 2. エフェクトシステム: インターフェース更新
```cpp
// 既存
virtual void process(ImageF32x4_RGBA& image) = 0;

// 新しいインターフェース (後方互換性維持)
virtual void process(VideoFrame& frame) {
    // デフォルト実装は既存のRGBAのみ処理
    process(frame.getRGBA());
}
```

### 3. ブレンドパイプライン: チャンネル別ブレンド
各チャンネルに適切なブレンドモードを適用するだけ:
- RGBA: 通常のアルファブレンド
- 深度: `min()` 演算
- 法線: 前面の値を上書き
- 速度: 加重平均
- ObjectId: 前面の値を上書き

## 💣 リスクと制約

⚠️ 全くないわけではないが最小限

| 項目 | リスク | 備考 |
|------|--------|------|
| メモリ使用量 | 🟡 中 | 8チャンネル有効時で 1920x1080 = 66MB/フレーム |
| GPU帯域 | 🟡 中 | メモリ帯域がボトルネックになる可能性 |
| 既存エフェクト互換性 | 🟢 小 | デフォルト実装で自動的に後方互換 |
| パフォーマンス低下 | 🟢 小 | 不要なチャンネルは動的に無効化可能 |
